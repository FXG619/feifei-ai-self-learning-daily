# Codex 工作流

## 主线程

- 使用一个 pinned durable thread 作为本项目主线程。
- 每次新增功能前先让 Codex 读取 `docs/` 和 `AGENTS.md`。
- 让 Codex 先创建 plan，再写代码。

## 修改后的验证

每次改完必须运行：

```bash
npm run build
npm run validate:content
```

如果当前还没有 `package.json`，说明无法运行，等站点初始化后再补跑。

## 日报生成与发布

- 每次生成日报只生成 `draft`，不自动发布。
- 飞飞审核后手动把 `status` 改为 `published`。
- 政策、考试、升学、心理、工具推荐类内容必须人工审核。

## 本地工作流

如果 GitHub App 权限不可用，就先使用本地工作流：

- 修改本地文件。
- 展示 diff。
- 由飞飞确认。
- 不擅自 push。
- 不擅自创建 PR。

## 预览反馈口径

如果在侧边栏预览页面发现问题，飞飞可以直接说：

- 这段太像机构文案。
- 这段家长看不懂。
- 今天只做一件事不够突出。
- 这个按钮太技术化。

## 后续可复用 Skill

后续可把“生成日报”“审核内容”“生成分享卡片”包装成可复用 skill。
