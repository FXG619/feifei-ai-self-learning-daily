# AGENTS.md

本文件是“飞飞的 AI 自学力日报”的长期协作规则。后续 Codex 任务开始前，应先读取本文件和 `docs/` 下的项目文档。

## 项目原则

- 不要把本项目做成泛教育新闻站。
- 不要把本项目做成 AI 工具导航。
- 不要让 AI 替孩子学习。
- 每篇内容必须帮助孩子多长出一点自学能力。
- 每篇内容必须能被中学生和家长看懂。
- 每篇内容必须有具体行动。
- 每篇内容必须强调“今天只做一件事”。

## 工程原则

- 第一版不要数据库。
- 不要登录系统。
- 不要复杂 CMS。
- 不要学生数据上传。
- 不要引入用户追踪。
- 不要提交 secrets。
- 不要自动发布高风险内容。

## 发布原则

- `draft` 永远不能公开展示。
- `published` 才能展示。
- RSS 只能包含 `published`。
- AI 生成的内容默认 `draft`。
- `source_digest` / `mixed` 内容默认至少 `medium` risk。
- 政策、考试、升学、心理、工具推荐内容必须人工审核。

## 修改后的验证

项目初始化完成后，每次修改必须运行：

```bash
npm run build
npm run validate:content
```

如果当前还没有 `package.json`，先不要运行这些命令，并在结果里说明：项目尚未初始化，下一步搭 Astro 后再运行。
